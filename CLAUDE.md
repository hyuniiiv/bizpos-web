## 1. 코딩 전에 먼저 생각하기

**가정하지 말고, 혼란을 숨기지 말 것.**

구현 전에:
- 가정은 명시적으로 밝힌다. 불확실하면 먼저 질문한다.
- 여러 해석이 가능하면 제시하고 선택을 맡긴다 — 조용히 혼자 결정하지 않는다.
- 더 단순한 방법이 있으면 말한다. 필요하면 반론한다.
- 무언가 불명확하면 멈추고, 무엇이 불명확한지 이름 붙여 질문한다.

---

## 2. 단순성 우선

**문제를 해결하는 최소한의 코드만. 추측성 코드 금지.**

- 요청하지 않은 기능은 추가하지 않는다.
- 단일 용도 코드에 추상화 레이어를 만들지 않는다.
- 요청하지 않은 "유연성"이나 "설정 가능성"을 추가하지 않는다.
- 발생 불가능한 시나리오에 대한 에러 핸들링을 추가하지 않는다.
- 200줄로 작성했는데 50줄로 가능하다면 다시 작성한다.

스스로 물어볼 것: "시니어 엔지니어가 이걸 보면 과도하게 복잡하다고 할까?" → Yes라면 단순화한다.

---

## 3. 수술적 변경

**꼭 필요한 것만 수정. 자신이 만든 것만 정리.**

기존 코드 편집 시:
- 인접한 코드, 주석, 포맷을 "개선"하지 않는다.
- 고장나지 않은 것을 리팩터링하지 않는다.
- 기존 스타일에 맞춘다 — 내가 다르게 하고 싶더라도.
- 무관한 데드 코드를 발견하면 언급만 한다 — 삭제하지 않는다.

내 변경으로 고아가 된 것들:
- 내 변경이 만든 미사용 import/변수/함수는 제거한다.
- 기존에 있던 데드 코드는 요청 없이 제거하지 않는다.

기준: 변경된 모든 줄이 사용자의 요청과 직접 연결되어야 한다.

## 4. 목표 중심 실행

**성공 기준을 먼저 정의하고, 검증될 때까지 반복한다.**

작업을 검증 가능한 목표로 변환:

## 5. BIZPOS 프로젝트 특화 규칙

### 빌드 필수

**파일 수정 후 반드시 `npm run build` 실행 및 결과 보고.**

### 결제 API 수정 시 극도 주의

`app/api/payment/` 하위 모든 파일은 실제 결제와 직결됩니다:
- `requireTerminalAuth` 누락 시 무인증 결제 가능 → 반드시 인증 체크
- `termId`는 항상 `auth.payload.termId` 사용 (바디 값 신뢰 금지)
- 취소/승인 API 변경 시 단건/일괄 양쪽 모두 검증

### 배포 순서 (보안 변경 포함 시)

1. Electron 앱 빌드·배포 (클라이언트 먼저)
2. 모든 단말기 업데이트 완료 확인
3. 서버(Vercel) 재배포

**서버를 먼저 배포하면 구 클라이언트가 크래시할 수 있음.**

### 오프라인 동기화 주의

`lib/txSync.ts`, `lib/onlineSync.ts` 수정 시:
- 오프라인 결제 데이터는 분실 불가 — 폴백이 데이터를 소실시키는지 반드시 확인
- `syncedIds`는 서버가 명시적으로 확인한 것만 처리

### 아키텍처 경계

| 환경 | 특이사항 |
|------|---------|
| Electron (로컬) | `window.electronAPI`, IndexedDB, `transactions.json` 사용 |
| Vercel (서버리스) | 메모리 싱글톤·파일시스템 공유 불가 — SSE/파일 기반 기능 주의 |

### Gate Hook 대응

편집 전 다음을 먼저 제시:
1. 이 파일을 import하는 파일 목록
2. 변경되는 public 함수/클래스
3. 데이터 파일 구조 (해당 시)
4. 사용자 지시 원문

## 프로젝트 스택 요약

- **Frontend**: Next.js 16 (App Router) + TypeScript + React 19
- **Backend**: Next.js API Routes
- **Database**: Supabase (PostgreSQL)
- **Offline**: IndexedDB + `transactions.json`
- **State**: Zustand (persist)
- **Auth**: bcryptjs PIN + JWT (`jose`)
- **Deployment**: Vercel (서버) + Electron (단말기)
- **PG**: BizPlay (AES256-CBC + HmacSHA256)
